Basics with Node.js-a Note-taking App

Posted on 2019-08-02

This article introduces basics with Node.js by building a note taking app.

1. Introduction to the App

This article is a note and a summary to the Node.js course I’m taking on Udemy.
This app is written in javascript with no GUI, so users must use it via command in terminal. The notes taken down by users are stored in json files.
Users can add notes, remove notes, list all notes and read the content of a note with the given title.

2. Tutorial to the App

In this section, I will explain the codes along with related technical problems and skills in two parts: the frontend user interface and the backend file management system.

2.1 User Interface: app.js

For users to use the app, for example, to add a note they must use command lines as such:

As we can see, this causes argv array to have as many as five elements. However, we only want the last two to be parsed into the only content in quotation marks. Therefore, we will install Yargs with npm, which is a Javascript development tool for package management. Yargs will process the argv array by automatically detecting the command and the arguments afterwards called the “builder”. In the argument, it detects the type of the arguments (eg. title, body) by searching for “–” and parse the type to the property of a yargs object. For example. below is a yargs object:

All the yargs command can be set up in the yargs.command object. Yargs identifies the third argument as the command. If it matches up the setup command in the object, it conducts the handler function. The “describe” is an optional description to the command. In this case, if we command the program to “add” something, we need to give it something, or there is no point of adding. Thus, I use the builder to specify what follows the command: I need a title and a body. I need to set their “demandOption” to be true, that is, users must provide a title and a body when they want to add a note. Their note should be in the form of “string”, so I put it to the “type”. Again, their “describe” is optional.
Then the last part is a handler function that is called if the command is entered and the builders are satisfied. Note that I am using an arrow function which basically is a shorthand for a standard function, making the codes look much tidier and simpler.
In our case, we want the backend to do something, say add a note, after the command “add” is called. This will be discussed in the next section. All we need to know for now is that I’m calling the backend program “notes.js” to conduct a method “addNote”, and passed in two arguments “argv.title” and “argv.body”.
Pay attention to this dot expression to argv. Thanks to Yargs that detects the elements in argv starting with “–”, we now can directly access the string part of the title or the body. Also remember to pass in “argv” in the handler function (I personally forget this for several times). Though it is a variable appeared the first time, yet Javascript is written in C++, and in C++, all the programs run in main.cpp where argv is one of the two parameters that get to be passed in.
Let’s get back to “note.js”. Like many other programming languages, if you want the program to use a method in other files, you will need to import them first. In JS, we use “require” instead of “import”. Since I am using methods in “note.js” and “yargs”, I require them at the beginning of our “app.js” by:

Note the two different ways of “require”. If you want to require another JS file, you start with “./“ which means “at the current folder”, and then you put on the file name. If you want to require a npm package, you only need to type in the name as a string. What “note.js” gives you is all the method it is set up to give you in the file. We will also discuss it in the later section.
Now that you are asking “note.js” for a few methods, you have to have a variable to hold them. Thus, you create two variables to hold a package of methods. The proper way of naming the variables is by naming them by the file name or the npm package name, just like I did, but it won’t do harm to the program if you name them otherwise.
