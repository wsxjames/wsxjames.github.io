Todo List App

Posted on 2019-08-19

This article introduces the steps of making a simple iOS Todo List app. It involves table view, permanent data storage, and dealing with multiple view controller.

1. Building a Tabbed App

The basic way of starting to build an app in Xcode is to build a single view app which only contains one view. In this case, we want to build an app that shows the todo list on one tab, and allow users to add tasks for themselves on the other tab. When we start a new project, we should select the tabbed app.

Once created, Xcode show the following to us. You may see three views: a Tab Bar Controller, the first view and the second view. The first and the second view controller controls the elements on the first and the second tabs. Of course we are not limited by the two tabs. To add more tabs, we can first add a view controller from library.

Then, we click the yellow button and control drag to the new view controller, and select the view controller option. As you can see, the new tab is automatically added onto the tab controller.

2. First View Controller: Todo List

We might want to change details from the tab bar from the first view controller. We can click onto it and go to the bar at right. Here I have changed the title of the bar item which shows bellow the bar element icon. Then we can put a table view onto it from the library. Remember to set the constraints to edge of the screen.

We first click on the table view and increase the prototype cell to 1 which means the rest of cells can all be made from it.

Then we click the prototype cell itself and under its attribute we name the identifier a name in string. I now name it as “Cell”.

Then we need to control drag from table view to the yellow button above, and select both options.

Next, we can go to the coding part. We need to inherit from two classes of table view which are the two options we selected above: UITableViewDelegate, UITableViewDataSource. The two parent classes brings in two functions. The first function returns the number of cells and the second returns a cell object with its style.

In the first view controller class, we put in a string array called items to store all the todo list. Thus, the number of cell is the number of items in the array. We return the number by items.count. Then we set the cell object in the second function by passing in two arguments: first, the style (which I use default here); second, reuseIdentifier (which should be the identifier name of the prototype cell, so swift knows which prototype cell to apply the style to). Then we set the text of the cell using cell.textLabel.text and the input is the index of string in the items array. We use indexPath.row to access the index. Finally we return the cell.
Then, we use viewDidAppear rather than viewDidLoad because the later one can be called for multiple times whenever we call the first tab. We set a constant variable itemsObject to get object with key “items” (this is similar to data storage in html5 where you set and get stuff–here we are getting stuff), and they are all stored as object. Then we parse the object into string array in the if statement to prevent crash. Finally we need to remember to let table to reload data. This corresponds to the setting method in the second tab which will be covered later.

Finally, we want to delete an item when we swipe on an item. We are using an editingStyle that if you type the word in Xcode, you can immediately see this pop up. Then we want to remove data from our items array, reload data and set the data into permanent data storage.

3. Second View Controller: Add Item

The UI of the second view controller is much better than the last one. All you need is just an outlet for textViewField and an action for a button for adding. The code for adding is rather important. All the codes will be executed when there is something in the text area. We first need to get what’s already in the permanent data storage and store that to a constant variable. Then we convert the itemsObject into a string array and store that into a variable item of larger scope if conversion is successful. If successful, we append the content of textField into the string array item, else we just put the text in the textField into the string array. Finally we set the string array to permanent data storage and clears up the textField.

Additionally we need some code for users to end their typing. The first method indicates that when users touch any place than the keyboard, we will close up the keyboard by endEditing(true). The second method is the method to control what happens when a user clicks onto the return statement. We can do various things with this method, like sending a message etc. However here we are just shutting down the keyboard by resignFirstResponder()
